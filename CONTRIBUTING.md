# Contributing

Thank you for contributing to Trugate documentation.

## Writing principles

- Use clear and precise language.
- Prefer short paragraphs over long blocks of text.
- Explain business concepts before technical details where relevant.
- Keep product claims aligned with actual platform scope.
- Avoid mixing future vision with current product functionality without labeling it clearly.

## Documentation standards

- Use sentence case headings.
- Use Markdown tables sparingly.
- Use Mermaid diagrams for core flows.
- Include request and response examples for API pages.
- Keep examples realistic and consistent with Trugate terminology.

## Pull requests

When submitting documentation changes:
- explain the reason for the change,
- note whether it affects product scope, terminology, or architecture,
- and flag any sections that may need legal or compliance review.
